The Process

From Data to Dialogue

Understanding the complete lifecycle of a Retrieval-Augmented Generation query.

1

Ingestion & Chunking

You upload a document via the frontend. Apache Tika extracts the raw text, and we split it into smaller, overlapping semantic chunks.

2

Vector Embedding

Each chunk is converted into an array of floats (vectors) representing its mathematical meaning. These are saved securely to PostgreSQL.

3

Semantic Query

When a user asks a question, we embed that query. The system finds the top chunks in the database whose vectors are mathematically closest to the query's vector.

4

LLM Generation

Our local Large Language Models read the combined context of the query and retrieved chunks, generating a fluid, natural response that is guaranteed to be accurate.
